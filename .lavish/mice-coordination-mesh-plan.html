MICE product direction · 22 July 2026
Decision: build a coordination substrate, not another agent launcher

Turn parallel worktrees into one aware team.

MICE should help incompatible coding agents share only the context that matters: work intent, dependency state, live change surfaces, decisions, and verified completion evidence. It should predict merge risk before review—not pretend it can know product intent from a filename.

RecommendationProceed only with a narrow, measured wedge: cross-harness activity capture → hunk-aware conflict forecasting → one actionable completion handoff. Worktree creation and a status board alone are table stakes.
Market reality

This is not an empty market. The unclaimed piece is the bridge.

Codex, Claude Code, GitHub Copilot, Conductor, and Vibe Kanban already run agents in parallel with isolated worktrees. Claude teams and Copilot subagents also provide same-vendor task/messaging primitives. Vibe Kanban and Conductor handle conflicts after integration has begun. The opportunity is not “many agents”; it is trustworthy coordination across tools and branches before human merge work accumulates.

Important correction to the earlier MICE research: “worktrees + status + shared task lists” is no longer a differentiator. The experiment must prove that MICE prevents integration work or saves human attention beyond these mature baselines.

Use the base, do not confuse it for the finished product

MICE already has a seed of this feature.

ExistsLocal MCP server

mice connect, mice mcp-server, and agent-facing tools already provide an opt-in integration point.

ExistsShared event memory

Append-only local events, agent subspaces, a query tool and a basic team_status view already live outside the repository.

GapAutomatic truth

Current overlap depends on agent-reported files, has no repository/worktree identity, no liveness model, and treats historical touches as active.

GapActionable delivery

There is no worktree registry, hunk-level predictor, inbox/adapter contract, completion state machine, or native coordination notification.

Verified in the current codebase: crates/mice-cli/src/memory.rs persists append-only events and reports file overlap; main.rs already exposes memory_note, memory_query, and team_status through MICE MCP. The next milestone must evolve these rather than create a duplicate store.

The product model

Shared project knowledge, private agent memory, scoped context cards.

“Everyone shares all memory” sounds attractive but creates context pollution, privacy problems and stale instructions. Instead, each agent keeps its own transcript/private context. MICE holds a small, repo-scoped, inspectable coordination ledger and constructs a bounded context card only when an agent joins, asks, hits a risk, or becomes newly available.

1. Identity and lifecycle

Register every worktree under its Git common directory, base ref and branch; record harness, parent agent, task, leases and heartbeats. A distinct clone/repository is not a merge domain.

2. Git is the source of truth

Read Git status, merge base, tracked paths, rename state and zero-context diffs. Agent claims add intent; Git observations prevent a stale or dishonest claim becoming truth.

3. Deliver only what applies

On every agent’s next MICE tool call, return its messages plus only relevant decisions, dependencies and change risks. Managed adapters may inject sooner; generic terminals cannot be truthfully “pushed” into.

Core interaction

The completion popup is a triage handoff, not a celebration.

  1. Launch or attach. mice team workspace create creates a named worktree/branch; mice team attach adopts an existing one. MICE never needs to own the agent process to observe Git.
  2. Observe and announce. A filesystem/Git watcher updates active changes. A voluntary adapter reports intent, task state, summary, parent/child relation and checks.
  3. Warn at the right precision. Same file is yellow only; intersecting changed line ranges against a shared merge base is red. Both agents receive a short, attributed risk message at the next supported delivery point.
  4. Finish with evidence. “Ready” requires a managed-process exit or agent completion event and a clean state/declared checks. Otherwise label it “possibly idle,” never “finished.”
  5. Offer one safe next action. Suggest an unclaimed, dependency-ready task with a declared or inferred disjoint surface. The human approves reassignment; MICE does not silently re-task an agent.

Merge intelligence

Predict, explain, and only then resolve—under explicit control.

Risk ladder

Green distinct paths or only generated/lock-file churn (suppressed by policy).

Yellow same file, but currently disjoint hunks or a rename/API dependency signal; show once, dedupe until the fact changes.

Red changed line intervals overlap relative to the same merge base, or a read-only merge simulation finds a textual conflict. Include branches, intervals, author intent and the safest next move.

What MICE must never claim

A file overlap is not automatically a merge conflict. A clean textual merge is not automatically a correct integration. Different repositories cannot have a Git merge conflict merely because paths match.

Automatic resolution is a separate, guarded operation: create a disposable integration worktree/branch, produce a proposed diff and test report, then require human confirmation before any target branch mutation.

Deterministic first pass

base = merge-base(branch_a, branch_b)
changes = diff(base → each worktree, including uncommitted)
classify(path, renamed_path, hunk_intervals, generated_policy)
if committed: read-only merge-tree simulation
if red: durable risk event + deduplicated inbox notice

Semantic assistance is advisory

After deterministic evidence says “red,” a model may summarize each side’s declared intent and draft resolution choices. It must cite live file ranges and never write its answer into persistent memory as source truth. Display relevant code ephemerally, not in MICE’s ledger.

Make adapters thin; make the coordination core vendor-neutral.

Delivery plan

Build the proof before the command center.

P0 · 1–2 weeks

Read-only shadow mode

Repo/worktree registry, Git observer, baseline event schema and a local report. No popups, no agent messaging, no merge mutation.

Exit: reliable worktree discovery and labeled historical conflict dataset.

P1 · 2–3 weeks

Predictive pilot

Hunk-aware risk engine, dedupe, activity context card and team_status_v2. Show risks in CLI/MCP only; compare predictions to real rebases/merges.

Exit: precision/recall threshold achieved in shadow data.

P2 · 2 weeks

Human attention loop

New IPC notification type and Swift native completion/risk popups. Add task leases, dependencies and deterministic next-safe-task recommendation.

Exit: completion handoff reduces human polling, without alert fatigue.

P3 · gated

Adapters and resolution

Codex/Claude connection adapters, delivery-capability UI, then an opt-in disposable integration-worktree resolution flow.

Exit: zero unsolicited branch mutations; successful proposals are reviewed and tested.

Architecture ownership stays aligned with MICE

Rust core

Registry, Git watcher/analysis, ledger, risk engine, task graph, MCP tools, and IPC commands. It owns facts and routing.

mice-ipc

New typed coordination.notify, coordination.openTeam, and action-response shapes—no duplicate Swift/Rust wire structs.

Notification Center permission/state, compact native panel, review/open/snooze actions. It renders; it does not schedule or choose routing.

The anti-gimmick test

Success is less integration rework and less human coordination time—not more telemetry.

Run this as a within-subject study: identical task bundles, models, token budgets, base commits and harness mix; alternate baseline worktree tooling (Codex/Conductor/Vibe/CLI) with MICE enabled. Begin observationally, then enable one intervention at a time. Instrument no source text, credentials, captures or transcript bodies.

Hard product boundaries

What keeps this useful instead of noisy or unsafe.

Non-goal: clone Conductor

MICE can create/attach worktrees, but should not lead with a kanban/PR product. Interoperate with those systems; own the cross-harness coordination facts.

Non-goal: global transcript pooling

Never persist raw diffs, commands, agent thought/transcripts, captures, clipboard or credentials in the coordination ledger. Use bounded summaries and Git-derived metadata with review/forget controls.

Non-goal: autonomous merge authority

Propose and test in a disposable integration worktree; only a human-confirmed action can alter a target branch. Keep evidence for every recommendation.

Positioning

The concise product promise

MICE is the local coordination memory for a heterogeneous coding team. It sees the Git truth across worktrees, gives each agent the minimum useful context, warns before changes become merge work, and returns human attention to the highest-leverage moment.
